An old Xorg is included, I plan to compile and package Gnome after Schillix is

> -what package management and update distribution/straegy does Schillix
> use when upgrading and distributing new applications etc, and what is
> the plan in this?

SVR4 packages seem to offer the best base, because:

-	they are compatible with previous versions

-	they support sparse zones

-	none of the new features seen with IPS cannot be  implemented
	using pkgadd (in special an installation over the network).
	Installation tests over the net show aprox. 10 minutes for the 
	500 MB base binaries.

-	pkgadd does not suffer from network latency or CD/DVD seek times,
	so an installation in India or one from DVD will be ready in
	less than a week ;-)

> -What might be the closest development needs/plans in Schillix, e.g.
> what eventually new people can/should do in contribution sense.

Since September 2010, Schillix has a state that allows cuncurrent independent 
development at nealy all places. There are several work packages, that are of 
interest:

-	compile and create packages for the SFW source consoliation.

-	compile and create packages for a more recent Xorg

-	compile and create packages for GNOME

-	convert Schillix to be based on SVR4 packages (ONNV packages are ready
	for a network install from berlios.de) 

-	convert the Schillix base (what you need to add to ONNV to make it 
	bootable without GUI) to SVR4 packages

-	Marketing ;-)

-	Add content to the web services

-	Community work
