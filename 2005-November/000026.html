Wajih Ahmed <wajih.ahmed at gmail.com> wrote:

> I want to customize Schillix 0.3 by adding my own application and then
> re-creating the iso.  I have looked at README-kit and have understood most
> of the requirements.  Is there a HOWTO or some other writeup which explains
> how to add additional packages to the base-kit?

Check the files "Packets-base" and "Packets-extension"

They cpontain lists of tar archives and star options used for RAM Disk 
extraction and general star options (e.g. needed in order to extract 
ideosyncratic packaging as in devpro-libm-bins-20051024.i386.tar.bz2).



> Without digging too deep it seems that i have to add my custom packaging by:
>
> 1.  Installing and configure the application (it is in Solairs package
> format) on a Solaris 9 or greater machine.

??? SVr4 packaging is not yet implemented in SchilliX-0.3 altough the packaging
software os present.

> 2.  Create a tar.bz2 file of the applicaiton including the startup scripts.

This is how the current tar based system works. Theck the content of the tmp/
dir in the packages for post install methods.

> 3.  Follow instructions in README-kit
> Add the name of the file to FILES and FILES-pkg
> 5.  Run

The files FILES and FILES-pkg are only used to create the 
archive schillix-0.3-base-kit.tar.gz2

NOTE: If you like to create your own tar archives either exclude all standard 
directories or directories with standard permissions at create time or
use something like the star option: "-find ! -type d" in the "Packets-base" and 
"Packets-extension" lines.
